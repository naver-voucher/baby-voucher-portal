## 0. 준비물

- 네이버 사내 계정 (Works/nDrive 접근)
- 공유 드라이브 `베이비바우처_파트너_운영` 편집 권한 (전임자가 부여)
- Google Sheet `BV_Partner_DB` 편집 권한
- dl_babyvoucher 메일링리스트 멤버 가입

위 4가지가 없으면 전임자에게 요청 후 진행.

---

## 1. admin 포털 열기 (1분)

1. 운영 리포 로컬 clone
   ```
   git clone <internal-repo-url>
   cd 바우처_파트너_포털
   python3 -m http.server 8765
   ```
2. 브라우저에서 `http://localhost:8765/admin.html`
3. 로그인 비밀번호: 전임자에게 문의 (또는 Works OAuth 연동 사용)

---

## 2. Apps Script URL 연결 (1분)

1. admin 포털 로그인 후 → 좌측 하단 설정
2. **Apps Script URL** 입력:
   ```
   https://script.google.com/macros/s/<DEPLOY_ID>/exec
   ```
   (OPERATIONS.md > "7. 자주 쓰는 URL·ID" 참고)
3. 저장 후 페이지 새로고침
4. 좌측 메뉴 **판매자 포털 > 제출 현황** 이 로드되면 연결 성공

---

## 3. 월별 ZIP 다운로드 테스트 (1분)

1. 좌측 메뉴 **판매자 포털 > 월별 아카이브**
2. 지난달 행의 `ZIP 다운로드` 클릭
3. 파일이 내려오면 OK → nDrive 월별 백업 폴더에 드래그

이게 **매월 25일의 유일한 수동 업무**다.

---

## 4. 판매자 링크 발송 테스트 (1분)

1. 좌측 메뉴 **판매자 포털 > 제출 현황**
2. 테스트 브랜드 1개 선택 → "링크 복사"
3. 복사된 URL 을 새 탭에서 열어 seller.html 이 정상 렌더링되는지 확인
4. 더미 데이터 제출 → admin 제출 현황에 뜨는지 확인
5. 확인 후 Submissions Sheet 에서 테스트 행 삭제
